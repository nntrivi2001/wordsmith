### Show-Don't-Tell for Mystery/Revelation
```
Thay vì: "Cô ấy rất sợ hãi"
Hãy viết: "Khuôn mặt của Endou trở nên tái mét hẳn đi, cơ thể hắn run bần bật..."

Pattern: Biểu hiện cụ thể thay vì nói trực tiếp
```

## I. Common Tropes (to avoid or subvert)

| Trope | Problem | Frequency |
|-------|---------|-----------|
| Protagonist has special ability to see through rules | Reduces reasoning difficulty | Extremely high |
| Rules always can be cracked | Lacks real threat | High |
| Teammates die one by one | Patterned death | High |
| Finally escape | Predictable ending | High |
| Rules source is evil god/curse | Single explanation | Medium |
| Protagonist is sole survivor | Lacks variation | Medium |

## II. Anti-Convention Constraints (select at least 1 per project)

### 2.1 Protagonist Ability Constraints

| ID | Anti-Trope Rule | Creative Driver | Example |
|----|----------------|-----------------|---------|
| RM-001 | Protagonist cannot see complete rules | Must use trial and error/reasoning | Rule manual has missing pages |
| RM-002 | Protagonist's special ability has serious side effects | Ability use requires trade-off | Seeing through rules causes memory loss |
| RM-003 | Protagonist's ability will be "learned" by rules | Same move can only be used once | Rules adapt to protagonist's cracking methods |
| RM-004 | Protagonist is part of the rules | Protagonist's actions affect rules themselves | Protagonist's death triggers new rules |
| RM-005 | Protagonist has no special abilities | Pure wisdom and observation | Ordinary person perspective |

### 2.2 Rules Itself Constraints

| ID | Anti-Trope Rule | Creative Driver | Example |
|----|----------------|-----------------|---------|
| RM-006 | Rules cannot be cracked, only adapted to | Survival not victory | Rules are world laws, cannot be changed |
| RM-007 | Rules evolve/mutate | Dynamic threat | Each death updates the rules |
| RM-008 | Rules conflict with each other | Use rules against rules | Following A rule inevitably violates B rule |
| RM-009 | Rules' punishment is delayed | Don't know when triggered | Violation triggers punishment at random time |
| RM-010 | Rules' true meaning differs from literal | Word games | "Don't talk" actually means "Don't lie" |

### 2.3 Environment/Team Constraints

| ID | Anti-Trope Rule | Creative Driver | Example |
|----|----------------|-----------------|---------|
| RM-011 | There's a mole on the team | Trust crisis | Someone deliberately triggering rules |
| RM-012 | Escaping is not the end | Rules follow | Rules take effect in real world after escape |
| RM-013 | Time loop | Repeat with cost | Death returns to starting point, but memory incomplete |
| RM-014 | Rules space expanding | Urgency | Rules area doubles every hour |
| RM-015 | Following rules also has cost | No safe option | Following rules slowly loses humanity |

### 2.4 Information Constraints

| ID | Anti-Trope Rule | Creative Driver | Example |
|----|----------------|-----------------|---------|
| RM-016 | Protagonist doesn't know they're in rules space | Discovering truth is first step | Thought it was normal world |
| RM-017 | Rule manual is fake/incomplete | Cannot fully trust information | Manual is a trap |
| RM-018 | Different people see different rules | Information asymmetry | Each person can only see part of the rules |
| RM-019 | Rules are forgotten | Memory management | Knows rules then gradually forgets |
| RM-020 | Speaking rules causes them to stop working | Silent game | Telling others rules changes them |

## III. Non-Trope Cool Points (replacing traditional escape)

### 3.1 Reasoning Cool Points

| ID | Cool Point Pattern | Description | Example |
|----|--------------------|-------------|---------|
| RM-S01 | Rule reverse reasoning | Infer rules from results | Infer rule content from death patterns |
| RM-S02 | Rule combination | Use combination effect of multiple rules | A rule + B rule = safe zone |
| RM-S03 | Rule boundary testing | Find precise boundary of rules | What's the boundary of "can't talk"? |
| RM-S04 | Rule source reasoning | Infer why rules exist | Rules are someone's obsession |
| RM-S05 | Fake rule identification | Identify which are real vs fake rules | Fake rules mixed into manual |

### 3.2 Game Theory Cool Points

| ID | Cool Point Pattern | Description | Example |
|----|--------------------|-------------|---------|
| RM-S06 | Rules against rules | Use rule conflicts | Use A rule's punishment to offset B rule |
| RM-S07 | Human nature game | Game with other survivors | Whoever violates first dies first |
| RM-S08 | Game with rule maker | Discover will behind rules | Rules are some existence's game |
| RM-S09 | Information game | Use information asymmetry | Pretend not to know rules |
| RM-S10 | Time game | Use rules' time characteristics | Complete goal before punishment triggers |

### 3.3 Reversal Cool Points

| ID | Cool Point Pattern | Description | Example |
|----|--------------------|-------------|---------|
| RM-S11 | Rule reversal | True meaning of rules revealed | "Don't open door" is to protect people outside |
| RM-S12 | Identity reversal | Protagonist/NPC identity revealed | Protagonist is rule's creator |
| RM-S13 | Goal reversal | Real goal is not escape | Escape causes greater disaster |
| RM-S14 | Timeline reversal | Truth of time sequence | This is the Nth loop |
| RM-S15 | Reality reversal | Relationship between rules space and reality | Rules space is the real world |

### 3.4 Cost Cool Points

| ID | Cool Point Pattern | Description | Example |
|----|--------------------|-------------|---------|
| RM-S16 | Sacrifice for information | Trade life for key information | Teammate passes rules before death |

## IV. Combination Examples

### Example 1: Information Asymmetry Mystery

```
Anti-trope rules: RM-018 (different people see different rules) + RM-020 (speaking rules causes them to stop working)
Non-trope cool points: RM-S09 (information game) + RM-S07 (human nature game)

Core conflict: Each person only knows part of the rules, but can't share
Creative driver: How to cooperate when can't communicate directly?
Differentiation: Social reasoning type mystery, focus on game between people
```

### Example 2: Rule Evolution Mystery

```
Anti-trope rules: RM-007 (rules evolve) + RM-003 (rules learn from protagonist)
Non-trope cool points: RM-S03 (boundary testing) + RM-S06 (rules against rules)

Core conflict: Each crack makes rules harder
Creative driver: How to survive when rules constantly evolve?
Differentiation: Dynamic threat, each loop is different
```

### Example 3: Cost-Type Mystery

```
Anti-trope rules: RM-015 (following rules also has cost) + RM-006 (rules unbreakable)
Non-trope cool points: RM-S17 (partial compliance) + RM-S20 (shared burden)

Core conflict: Both following and violating have costs, can only choose which cost to bear
Creative driver: How to minimize cost?
Differentiation: No safe option, every choice has weight
```

### Example 4: Identity Mystery

```
Anti-trope rules: RM-004 (protagonist is part of rules) + RM-016 (protagonist doesn't know in rules space)
Non-trope cool points: RM-S12 (identity reversal) + RM-S15 (reality reversal)

Core conflict: Protagonist discovers they are the core of the rules
Creative driver: How does protagonist face truth of being a "monster"?
Differentiation: Protagonist perspective subversion, from victim to perpetrator
```

## V. Rules Design Template

### 5.1 Basic Rules Template

```markdown
【Rule Name】
Trigger condition: {What triggers the rule}
Rule content: {Specific requirements of the rule}
Violation punishment: {Consequences of violating rules}
Hidden meaning: {True purpose/deep meaning of rules}
Exploitable points: {Loopholes/boundaries protagonist can exploit}
```

### 5.2 Rules Design Principles

1. **Inherent logic**: Rules must have reason to exist
2. **Reasonable**: Readers should be able to infer rules
3. **Clear boundaries**: Rule boundaries should be testable
4. **Clear cost**: Violation punishment should be specific
5. **Exploitable**: Rules should have exploitable space

### 5.3 Rules Complexity Control

| Length | Core Rules | Hidden Rules | Fake Rules |
|--------|-----------|--------------|------------|
| Short (under 50K words) | 3-5 | 1-2 | 0-1 |
| Medium (50K-200K words) | 5-8 | 2-3 | 1-2 |
| Long (200K+ words) | 8-12 | 3-5 | 2-3 |

## VI. Usage Guide

### 6.1 Selection Process

1. Select 1-2 from "Anti-Convention Constraints"
2. Select 2-3 types from "Non-Trope Cool Points" as main cool point patterns
3. Design 3-5 core rules
4. Design "exploitable points" for each rule
5. Check if there are interesting interactions between rules

### 6.2 Avoiding Pitfalls

- ❌ Rules too easily cracked
- ❌ Rules punishment not terrifying enough
- ❌ Protagonist ability too strong, rules meaningless
- ❌ Rules unrelated to each other
- ✅ Rules have real threat
- ✅ Cracking requires cost
- ✅ Interesting interactions between rules
- ✅ Protagonist's victory comes from wisdom not ability

## Vietnamese Writing Patterns (STYLE_GUIDE_VN.md Section 13)

### From Ta Dung Nhin Vo Lam Toang (Primary Source):
- Dialogue: "Nội dung" + action tag, no ——
- Inner thoughts: third-person narrative, "cậu" for self-reference
- Scene breaks: --- for major, *— Hết Chương —* for end
- First-person: cậu/mình in thoughts, tao/mày with close people
- Slang: vãi, cứt, bro (@ v @) in GenZ contexts

### Sentence Structure:
- Must have Subject + Predicate
- Connectors: và, nhưng, nên, vì, sau đó, rồi, thì, mà
- No fragmented sentences

### 8 Error Types:
1. Units: mét/cm/km/kg (NOT trượng/dặm/tấc/thốn/ly)
2. Vocabulary: match context
3. Punctuation: — single (NOT —— double)
4. Sentence: complete with subject+predicate
5. Spelling: ken két not kẽ kẽy
6. Connectors: required
7. Subject: explicit in descriptions
8. Natural Vietnamese words

### Cool-Point Placement:
- Every 500-800 words
- Fast-paced: 70-80% short sentences (1-7 words)
- Slow-paced: 70-80% long sentences (15+ words)

### Cliffhanger Patterns:
- Incomplete action ending
- System notification: "Keng!" + message
- Twist revelation
- Question left unanswered
