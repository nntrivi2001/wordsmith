# Rules Mystery / Honkaku Genre Template

> **Core Appeal**: Logical reasoning + Fair clues + Intellectual gaming. Readers and detectives stand at the same starting line, truth must be derivable.

---

## Creative Constraints (Optional)
- Recommended Pack: Pack M14 / Pack M06
- Universal Overlay: Pack U02

## 1. Core Sub-Genres

### Classic Deduction Flow
- **Feature**: Pure logical reasoning, locked room/tricks/alibi
- **Core Satisfaction**: Puzzle satisfaction, truth revealed, culprit punished
- **Typical Mode**: Case occurs → Investigate → Reasoning reveal

### Rules Mystery Flow
- **Feature**: Urban legends, bizarre rules, survival puzzle
- **Core Satisfaction**: Discover rules, use rules, escape alive
- **Typical Mode**: Encounter urban legend → Discover rules → Use rules to escape

### Suspense Thriller Flow
- **Feature**: Atmosphere creation, psychological fear, continuous reversals
- **Core Satisfaction**: Tension and excitement, truth shock, endless aftertaste
- **Typical Mode**: Suspense opening → Layer by layer → Shocking reversal

### Social Deduction Flow
- **Feature**: Explore criminal motives, critique social issues
- **Core Satisfaction**: Human depth, social reflection, emotional resonance
- **Typical Mode**: Behind case → Human excavation → Social critique

## 2. Classic Deduction Ten Commandments (Web Novel Adapted)

```
Commandment 1: Criminal must appear early in story
Commandment 2: Supernatural power forbidden for truth explanation (but can be used for misdirection)
Commandment 3: Locked room can have at most one secret passage
Commandment 4: Unknown poisons or devices requiring lengthy scientific explanations are forbidden
Commandment 5: Completely unknown mysterious person as murderer not allowed
Commandment 6: Detective cannot solve case by accident or sixth sense
Commandment 7: Detective cannot be the criminal (unless dual detective setting)
Commandment 8: Detective must publicly disclose all clues to readers
Commandment 9: Assistant's (Watson role) thoughts must be transparent to readers
Commandment 10: Twins/doubles must be indicated early
```

## 3. Four Core Pillars

### Pillar 1: Fair Play
```
✅ Correct example:
Chapter 10: Detective discovers vase fragments (reader sees simultaneously)
Chapter 20: Detective deduces "vase was broken from inside"
→ Reader can review and verify

❌ Wrong example:
Chapter 10: Detective discovers a clue (content not described)
Chapter 20: Detective says "I discovered key evidence long ago!"
→ Reader cannot verify, unfair
```

### Pillar 2: Logic
```
Logic chain example:

Known clues:
1. Victim in locked room
2. Window locked from inside
3. Victim holding key

Reasoning process:
1. Victim holding key → May have locked door themselves
2. Window inner lock → Murderer didn't escape through window
3. Room locked → Murderer may have left before victim locked door
4. Guess: Victim locked door after murderer was already inside
5. Verify: Look for places in room that could hide someone
```

### Pillar 3: Solvability
```
Clue distribution principle:
- Key clues appear at least 3 times (different forms)
- Red herrings cannot exceed number of true clues
- Must have "challenge to reader" before reveal
```

### Pillar 4: Surprise
```
Reversal levels:
Level 1: Murderer identity reversal
Level 2: Motive reversal
Level 3: Case nature reversal
Level 4: Narrator/perspective reversal
```

## 4. Rules Mystery Framework

### Rule Design Principles
```
1. Rules must have internal logic
2. Exceptions to rules need foreshadowing
3. Using rules to escape must be reasonable
4. Rule sources need explanation (can be mysterious but must hint)
```

### Rules Mystery Structure
```
Encounter phase (20%):
- MC enters anomalous environment
- Discover first victim/anomaly
- Obtain "rule manual" or verbal hints

Exploration phase (40%):
- Test rule boundaries
- Someone violates rules and is punished
- Gradually understand rule system

Utilization phase (30%):
- Discover rule loopholes
- Formulate escape plan
- Accidents during execution

Ending phase (10%):
- Successfully escape or fail
- Final rule explanation
- Aftertaste (open hint)
```

### Rule Examples
```markdown
【Elevator Rules】
1. Can only press floor you want to go to
2. Cannot speak when someone is in elevator
3. If you see extra person in mirror, close eyes immediately
4. When elevator stops at 4th floor, don't get off

【Rule Utilization】
- Use "cannot speak" to signal partner danger
- Use "mirror" to discover disguised person
- Use "4th floor forbidden" to lure enemy
```

## 5. Outline Structure (400K Words Standard)

### Volume 1: Case Occurs (Chapters 1-30, 15%)
- **Opening hook**: Body/anomaly/missing person
- **Main characters appear**: Detective, suspects, victim's relations
- **Environment setting**: Locked space/time limit/special rules
- **First investigation**: Basic clue collection

### Volume 2: Investigation (Chapters 31-100, 35%)
- **Suspect inventory**: Each person's motive, opportunity, alibi
- **Clue accumulation**: Physical evidence, witness testimony, intelligence
- **Misdirection**: Red herring appears
- **Small climax**: Second case/new discovery

### Volume 3: Deduction (Chapters 101-150, 25%)
- **Clue integration**: Correlate analysis
- **Hypothesis verification**: Elimination/confirmation
- **Truth approaching**: Key clues surface
- **Challenge to reader**: Final hint

### Volume 4: Reveal (Chapters 151-180, 15%)
- **Detective's reasoning**: Complete logical chain
- **Culprit revealed**: Identity confirmed
- **Motive revealed**: Crime reason
- **Trick explanation**: Method detailed

### Volume 5: Wrap Up (Chapters 181-200, 10%)
- **Culprit punished**: Legal/moral judgment
- **Character endings**: Each person's fate
- **Aftertaste**: Reflection/new case hint

## 6. Clue Design Techniques

### Three-Layer Clue Method
```
First layer: Surface clues (everyone can see)
- Death time, location, method
- Obvious physical evidence

Second layer: Hidden clues (need attention to discover)
- Casual details
- Contradictory statements
- Timeline gaps

Third layer: Deep clues (need reasoning/professional knowledge)
- Behavioral psychology analysis
- Professional field knowledge
- Historical/background connections
```

### Red Herring Design
```
Type 1: False suspect
- Has motive but no opportunity
- Present but has alibi

Type 2: False evidence
- Forged physical evidence
- Manipulated witness

Type 3: False reasoning
- Assistant's wrong deduction
- Detective's deliberate misdirection

Type 4: False identity
- Pretender
- Impersonator
```

## 7. Suspect Management

### Suspect Design Template
```
Name: ___
Relationship: Relationship with victim
Motive: Why kill
Opportunity: Whether has time to commit crime
Means: Whether has ability to commit crime
Alibi: Has/no/suspicious
Secret: Hidden information (not necessarily related to case)
```

### Suspect Quantity Suggestions
```
Short story (under 50K words): 3-5 people
Medium story (50K-200K words): 5-8 people
Long story (over 200K words): 8-12 people
```

## 9. Classic Trick Categories

### Locked Room Tricks
```
1. Mechanical lock (timing/remote/automatic)
2. Enter and hide in advance
3. Victim locks door after murderer then killed
4. Room not actually locked (illusion)
5. Death time falsified
```

### Alibi Tricks
```
1. Time falsified (clock/video)
2. Double substitution
3. Twins/similar person
4. Remote killing
5. Accomplice cooperation
```

### Identity Tricks
```
1. Dead person not really dead (disguise death)
2. Murderer not really murderer (framed)
3. Detective is the murderer
4. Narrator deceives
```
