# Zhihu Short Story Plot Compression Techniques

> The art of short stories is the art of deletion. This document provides practical techniques for compressing long-form plots into short stories.

---

## Vietnamese Writing Patterns

### From Ta Dung Nhin Vo Lam Toang & Sources:
- Dialogue format: "Content" + action tag
- Inner thoughts: third-person narrative without quotes
- Scene breaks: *** major, —0o0— minor
- Pronouns: mày/tao (close), tôi/ngài (formal), hắn (antagonist)
- First-person internal: cậu/mình

### 8 Error Types to Avoid:
1. Units: mét/cm/km/kg (NOT trượng/dặm/tấc/thốn)
2. Vocabulary: context-appropriate
3. Punctuation: — single (NOT ——)
4. Sentence: complete (subject + predicate)
5. Spelling: natural Vietnamese
6. Connectors: và/nhưng/nên/vì/rồi/thì
7. Subject: explicit in descriptions
8. Natural rhythm

### Genre-Specific Patterns:

#### Xuanhuan/Cultivation:
- Use mét/cm/km for distances (NOT trượng)
- Power levels: Trúc Cơ, Kim Đan, Nguyên Anh
- Dragon descriptions from Bien Nien Su Rong style

#### Isekai/Gaming:
- Status card format with "Keng!" notifications
- Combat: short → explanation → short punch
- Level-up pacing with 5% recovery chance

#### Romance/Fire Strand:
- Show-don't-tell emotion markers
- Building slow, conflict before resolution
- Body language over telling emotions

#### Rules-Mystery:
- Clue placement patterns
- Revelation timing
- Suspense building

## 1. Core Compression Principles

### Subtraction Thinking
```
Long-form thinking: This plot point can be added in
Short-form thinking: Should this plot point be deleted?
```

### Compression Formula
```
Short story = Core conflict + Key turning point + Emotional climax
Delete = Subplots + Transitions + Repetition + Explanations
```

---

## 2. Five Major Compression Techniques

### 1. Time Jump Method
```
Original (500 characters):
Detailed description of how female lead discovered husband cheating, tracked evidence, inner struggle...

Compressed (50 characters):
"Three months of tracking and evidence gathering resulted in a stack of photos and a divorce agreement."
```

### 2. Result First Method
```
Original: First setup cause, then write result
Compressed: Give result first, use flashback to supplement cause

Example:
"When he knelt in front of me, I remembered three years ago, he also knelt like this—proposing to another woman."
```

### 3. Dialogue Instead of Narration
```
Original (200 characters):
She knew her mother-in-law always looked down on her, thinking she wasn't worthy of her son...

Compressed (50 characters):
"My son is what kind of status, you are what kind?" Mother-in-law's words, she listened to for three years.
```

### 4. Detail Instead of Description
```
Original (300 characters):
Detailed description of how poor she is, how broken the house she lives in, how bad the food she eats...

Compressed (30 characters):
In the fridge there was only half a cabbage, she had been eating it for a week.
```

### 5. Blank Space Hint Method
```
Original: Detailed explanation of entire story
Compressed: Only give key information, let readers infer

Example:
"When getting DNA tested, she didn't cry. When results came out, she laughed."
(Don't explain why she laughed, let readers understand themselves)
```

---

## 3. Deletion Priority

### Must Keep
```
1. Core conflict (reason the story exists)
2. Main turning point (key to plot advancement)
3. Emotional climax (reader thrill point/pain point)
4. Ending twist (story's landing point)
```

### Delete First
```
1. Subplots of secondary characters
2. Repeated emphasized information
3. Transitional scenes
4. Background explanations
5. Repeated psychological activities
```

### Can Compress
```
1. Development with large time span
2. Repetition of similar plot points
3. Environment atmosphere description
4. Character relationship introductions
```

## 4. Common Plot Compression Examples

### Wife Pursuit Cremation Compression
```
Long version (500k characters):
Volume 1: Meeting and falling in love
Volume 2: Misunderstanding and heartbreak
Volume 3: Separation
Volume 4: Pursuing wife
Volume 5: HE

Short version (8000 characters):
Opening: Already separated, male lead pursuing wife (500 characters)
Flashback: How he abused her back then (2000 characters)
Present: Her change and coldness (2000 characters)
Turn: Truth revealed (2000 characters)
Ending: Broken mirror reunited (1500 characters)
```

### Rebirth Revenge Compression
```
Long version (800k characters):
Detailed rebirth planning, dealing with enemies one by one

Short version (5000 characters):
Opening: Rebirth moment + core enemy locked in (500 characters)
Planning: One key move (1500 characters)
Execution: Direct confrontation with big Boss (2000 characters)
Ending: Revenge success (1000 characters)
```

### Wealthy Family Drama Compression
```
Long version: Complex family relationships and power struggles

Short version (10000 characters):
Focus on one core conflict (true/fake daughter/inheritance dispute)
Keep only 2-3 key characters
One key confrontation determines victory
```

## 5. Information Delivery Techniques

### 1. Embedded Background
```
Wrong: First use 500 characters to introduce background
Correct: Naturally bring out in dialogue/action

Example:
"Three years already, you still live in this broken rental place?"
(One sentence brings out: time span + economic situation + character relationship)
```

### 2. Iceberg Writing
```
Only write the tip of the iceberg, let readers fill in the underwater part

Example:
"She glanced at her phone, the 47th missed call."
(Don't explain who called, why she didn't answer, readers understand themselves)
```

### 3. Symbolized Expression
```
Use one symbol to replace large paragraph of description

Examples:
- Ring: Marriage status
- Pregnancy test: Pregnancy
- Medical record: Serious illness
- Paternity test: Identity
```

## 6. Compression Checklist

### Deletion Check
- [ ] Is there a secondary character that can be deleted?
- [ ] Is there repeated information?
- [ ] Is there a transitional scene that can be skipped?
- [ ] Is there excessive psychological description?

### Efficiency Check
- [ ] Does each scene advance the plot?
- [ ] Does each dialogue carry information?
- [ ] Is information delivered with minimum words?

### Completeness Check
- [ ] Is the core conflict clear?
- [ ] Is the turn powerful enough?
- [ ] Does the ending have aftertaste?
